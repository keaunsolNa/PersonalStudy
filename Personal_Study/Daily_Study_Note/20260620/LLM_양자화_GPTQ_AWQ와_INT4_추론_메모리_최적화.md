# LLM 양자화 GPTQ·AWQ와 INT4 추론 메모리 최적화

> 2026-06-20 신규 주제 · 확장 대상: AI_Multi-Agent_서비스_실전프로젝트

## 학습 목표

- 양자화가 가중치 비트 폭을 줄여 메모리·대역폭을 절감하는 원리를 수식으로 설명한다
- GPTQ 의 헤시안 기반 오차 보정과 AWQ 의 활성값 인지 스케일링을 구분한다
- 가중치 전용 양자화와 W8A8 같은 활성값 포함 양자화의 적용 상황을 나눈다
- INT4 추론의 정확도 손실과 처리량 이득을 정량적 관점으로 평가한다

## 1. 왜 양자화인가: 메모리가 병목이다

LLM 추론에서 토큰을 하나 생성하려면 모델의 모든 가중치를 메모리에서 읽어 곱셈-누적을 수행해야 한다. 디코딩 단계는 배치가 작을 때 연산보다 **메모리 대역폭** 에 묶인다(memory-bound). 70B 모델을 FP16 으로 두면 가중치만 140GB 라 단일 GPU 에 올라가지도 않는다. 양자화는 가중치를 16비트에서 8/4비트로 줄여, 메모리 점유와 매 토큰마다 읽어야 할 바이트 수를 절반·1/4 로 낮춘다. 메모리-바운드 구간에서 읽을 바이트가 1/4 이 되면 처리량은 그에 비례해 올라간다. 즉 양자화의 1차 효과는 "모델을 올릴 수 있게" 하고, 2차 효과는 "더 빠르게" 만드는 것이다.

## 2. 양자화의 기본 수식

양자화는 연속적 부동소수 값을 정수 격자에 사상한다. 대칭 양자화의 핵심은 스케일 `s` 다.

```
s = max(|W|) / (2^(b-1) - 1)        # b 비트, 예: 4비트면 분모 = 7
W_q = round(W / s)                   # 정수로 양자화
W_dequant = W_q * s                  # 역양자화(추론 시 복원)
오차: ΔW = W - W_dequant
```

비대칭(affine) 양자화는 zero-point `z` 를 두어 `W_q = round(W/s) + z` 로 비대칭 분포를 더 잘 표현한다. 양자화 단위(granularity)도 정확도를 좌우한다. per-tensor 는 테서 하나에 스케일 하나라 거칠고, per-channel(출력 채널별), **group-wise**(예: 128개 가중치마다 스케일)가 더 정밀하다. INT4 양자화는 보통 group size 64~128 의 group-wise 를 쓴다. 그룹이 작을수록 정확하지만 스케일 메타데이터 오버헤드가 커진다.

## 3. 이상치(outlier) 문제

단순 반올림 양자화가 LLM 에서 무너지는 이유는 **활성값의 이상치** 때문이다. 트랜스포머의 일부 채널은 다른 채널보다 수십 배 큰 값을 가진다. 이 큰 값이 양자화 스케일을 지배하면, 대다수 작은 값들이 몇 개의 정수 격자로 뭉개져 정보가 사라진다. 그래서 LLM.int8() 은 이상치 채널만 FP16 으로 분리 계산하는 혼합 정밀도를 썬다. 하지만 이 분리는 커널 효율을 떨어뜨린다. GPTQ 와 AWQ 는 이상치를 다루는 더 효율적인 방법을 각각 제시한다.

## 4. GPTQ: 헤시안 기반 오차 보정

GPTQ 는 OBQ(Optimal Brain Quantization) 계열로, "가중치를 하나씩 양자화하되, 그로 인한 출력 오차를 아직 양자화하지 않은 나머지 가중치로 보상" 한다. 레이어의 출력 오차를 최소화하는 문제를 2차 근사하면, 입력의 헤시안 `H = 2 X Xᵀ` 가 등장한다.

```
목표: argmin_{W_q} || W X - W_q X ||²
한 가중치 w_q 를 양자화한 뒤, 나머지 가중치를 다음으로 갱신:
  δ = -(w - w_q) / [H⁻¹]_qq * H⁻¹_:,q   (양자화 오차를 남은 가중치에 전파)
```

직관은 이렇다. 어떤 가중치를 반올려 생긴 오차를, 그것과 상관관계가 큰 다른 가중치들을 조금 움직여 상쇄한다. 헤시안 역행렬이 그 상관 구조를 담는다. GPTQ 는 이를 열 단위로 효율화하고 Cholesky 분해로 수치 안정성을 확보해, 수백 개 보정 샘플만으로 수십억 파라미터를 몇 시간 내 4비트로 양자화한다. 결과적으로 INT4 에서도 perplexity 손실이 작다. 단점은 보정 데이터에 다소 민감하고, 활성값 양자화는 다루지 않는 가중치 전용 기법이라는 점이다.

## 5. AWQ: 활성값 인지 가중치 스케일링

AWQ(Activation-aware Weight Quantization)의 통찰은 "모든 가중치가 똑같이 중요하지 않다" 이다. 활성값이 큰 채널에 곱해지는 가중치(salient weight)는 출력에 더 크게 기여하므로, 이들의 양자화 오차를 줄이면 전체 손실이 크게 준다. AWQ 는 이 중요한 채널의 가중치를 양자화 전에 스케일 `s` 로 키우고(그만큼 활성값은 `1/s` 로 줄여 수학적 등가 유지), 키워진 가중치는 같은 격자에서 상대 오차가 작아진다.

```
원래:  Y = W · X
변형:  Y = (W · diag(s)) · (diag(1/s) · X)   # 수학적으로 동일
       ↑ 중요 채널 가중치를 s>1 로 확대 후 양자화 → 상대 오차 감소
스케일 s 는 활성값 크기 통계로부터 그리드 서치로 결정
```

AWQ 는 헤시안 역행렬 계산이 없어 GPTQ 보다 가볍고 빠르며, 보정 데이터 분포에 덜 민감해 일반화가 좋다는 것이 논문의 주장이다. 또한 가중치만 재배치하므로 추론 시 추가 연산이 없다. 실무에서 GPTQ 와 AWQ 는 둘 다 4비트 가중치 전용 양자화의 표준 선택지이며, 모델·태스크에 따라 우열이 갈려 둘 다 시도해 보는 것이 일반적이다.

## 6. 가중치 전용 대 활성값 포함 양자화

| 방식 | 표기 | 대상 | 특징 |
|---|---|---|---|
| 가중치 전용 | W4A16 | 가중치 4비트, 활성값 16비트 | 메모리·대역폭 절감, 디코딩(메모리바운드)에 유리 |
| 활성값 포함 | W8A8 | 가중치·활성값 8비트 | INT8 텐서코어로 연산량까지 절감, prefill(연산바운드)에 유리 |
| FP8 | E4M3/E5M2 | 8비트 부동소수 | 이상치 표현이 정수보다 부드러움, 최신 GPU 지원 |

W4A16(GPTQ/AWQ)은 토큰을 하나씩 만드는 디코딩 단계에 최적이다. 이 단계는 메모리-바운드라 가중치만 줄여도 큰 이득이다. 반면 긴 프롬프트를 한 번에 처리하는 prefill 이나 큰 배치는 연산-바운드라, 활성값까지 INT8 로 낮춰 텐서코어 정수 연산을 쓰는 W8A8(예: SmoothQuant)이 유리하다. SmoothQuant 는 AWQ 와 비슷하게 활성값 이상치를 가중치로 "이동" 시켜 활성값도 양자화 가능하게 만든다.

## 7. KV 캐시 양자화

가중치만이 메모리를 먹는 게 아니다. 긴 컨텍스트에서는 **KV 캐시** 가 가중치를 능가하기도 한다. 시퀀스 길이·배치·레이어에 비례해 커지기 때문이다. KV 캐시도 INT8/INT4 로 양자화하면 더 긴 컨텍스트와 큰 배치를 같은 메모리에 담을 수 있다. 다만 K/V 는 토큰마다 동적으로 생성되므로 per-token 또는 per-head 스케일로 다뤄야 정확도가 유지된다. 키(K)가 값(V)보다 양자화에 민감한 경향이 있어, K 는 더 높은 비트로 두는 비대칭 전략이 쓰이기도 한다.

## 8. 정확도-처리량 트레이드오프 평가

양자화를 도입할 때는 두 축을 함께 본다. **정확도** 는 perplexity(언어모델링 손실)와 다운스트림 벤치마크 정확도로 측정한다. 경험적으로 8비트는 FP16 대비 손실이 거의 없고, 4비트(GPTQ/AWQ)는 대형 모델에서 손실이 작지만 소형 모델일수록 손실이 두드러지고, 3비트 이하는 정교한 기법 없이는 급격히 무너진다. **처리량** 은 메모리-바운드 디코딩에서 비트 폭에 거의 비례해 향상되지만, 역양자화(dequant) 오버헤드와 커널 구현 품질에 따라 이론치에 못 미칠 수 있다. 따라서 도입 결정은 "목표 정확도 임계선을 지키는 가장 낮은 비트 폭" 을 실제 태스크 데이터로 검증해 고르는 것이 원칙이다. 같은 INT4라도 라이브러리(vLLM, TensorRT-LLM, llama.cpp 의 GGUF k-quant 등)마다 커널 효율과 정확도가 달라, 배포 대상 런타임에서 직접 벤치마크하는 절차가 필수다.

## 9. PTQ 대 QAT

지금까지는 학습 후 양자화(PTQ, Post-Training Quantization)를 다뤄다. GPTQ·AWQ 모두 이미 학습된 가중치를 소량의 보정 데이터로 양자화하는 PTQ 다. 반대편에 **양자화 인지 학습(QAT, Quantization-Aware Training)** 이 있다. QAT 는 학습 과정에 가짜 양자화(fake quantization) 노드를 삽입해, 순전파에서 양자화-역양자화를 거친 값으로 손실을 계산하고 역전파한다. 모델이 양자화 오차에 적응하도록 학습되므로 더 낮은 비트에서도 정확도가 잘 유지된다. 트레이드오프는 명확하다. QAT 는 정확도가 좋지만 전체 학습 파이프라인과 데이터셋, 큰 연산이 필요하다. 수십억 파라미터 LLM 을 QAT 로 다시 학습하는 것은 비용이 막대해, 실무에서는 PTQ 가 압도적으로 많이 쓰인다. 다만 4비트 미만 극단적 압축이나 정확도가 결정적인 도메인에서는 QAT 또는 그 중간 형태인 양자화 인지 파인튜닝(예: QLoRA 의 NF4 양자화 + LoRA 어댑터 학습)이 선택된다. QLoRA 는 베이스 모델을 4비트로 동결하고 작은 LoRA 어댑터만 학습해, 단일 GPU 에서 대형 모델 파인튜닝을 가능케 한 대표 사례다.

## 10. 양자화 포맷과 런타임

같은 INT4 라도 저장 포맷과 추론 런타임에 따라 정확도·속도가 갈린다. 대표 생태계를 정리하면 다음과 같다.

| 포맷/런타임 | 주 사용처 | 특징 |
|---|---|---|
| GPTQ(.safetensors) | vLLM, TGI, ExLlama | GPU 추론, group-wise INT4 |
| AWQ | vLLM, TGI | GPU 추론, 활성값 인지, 빠른 양자화 |
| GGUF(k-quant) | llama.cpp | CPU/GPU 혼합, 블록별 다양한 비트(Q4_K_M 등) |
| bitsandbytes(NF4) | Transformers, QLoRA | 파인튜닝 친화, on-the-fly 양자화 |

llama.cpp 의 **k-quant** 는 한 모델 안에서 레이어 중요도에 따라 비트를 섞는다. 예컨대 Q4_K_M 은 대부분 4비트지만 attention 의 일부 텐서는 더 높은 비트로 두어 정확도를 지킨다. 이런 혼합 비트 전략은 "전부 4비트" 보다 같은 평균 비트에서 정확도가 높다. 배포 시에는 타겟 하드웨어(서버 GPU 인지, 엣지 CPU 인지)와 런타임이 지원하는 포맷을 먼저 정하고, 그 위에서 양자화 기법을 고르는 순서가 합리적이다.

## 11. 도입 체크리스트

양자화를 프로덕션에 적용할 때의 점검 순서는 이렇다. 첫째, **목표를 정의** 한다 — 메모리를 줄여 모델을 올리는 것이 목적인지(W4A16), 처리량을 높이는 것이 목적인지(W8A8, 배치 큰 prefill)에 따라 방식이 갈린다. 둘째, **보정 데이터를 대표성 있게** 고른다. 실제 서비스 트래픽과 분포가 비슷한 수백 샘플이면 충분하며, 분포가 어긋나면 GPTQ 의 오차 보정이 엉뚝한 방향으로 최적화된다. 셋째, **실제 태스크로 정확도를 검증** 한다. perplexity 만 보지 말고 다운스트림 정확도(서비스가 쓰는 실제 평가셋)로 손실을 측정한다. 넷째, **타겟 런타임에서 처리량을 벤치마크** 한다. 이론적 비트 절감이 실측 속도로 이어지는지는 커널 구현에 달려 있다. 다섯째, **KV 캐시 양자화를 별도 검토** 한다. 긴 컨텍스트 서비스라면 가중치보다 KV 캐시가 메모리 병목일 수 있다. 이 다섯 단계를 거치면 "무작정 4비트로 줄였다가 정확도가 망가지는" 흔한 실패를 피할 수 있다. 핵심은 양자화가 공짜 점심이 아니라 정확도-효율의 명시적 거래라는 사실을 측정으로 통제하는 것이다.
