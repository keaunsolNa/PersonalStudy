# 개념
- CDN(Content Delivery Network)는 데이터 사용량이 많은 애플리케이션의 웹 페이지 로드 속도를 높이는 상호 연결된 서버 네트워크다.
- 사용자가 방문하려는 웹 서버에서 멀리 떨어져 있는 경우, 동영상 또는 웹 사이트 이미지와 같은 대용량 파일을 로드하는 데 오랜 시간이 필요하다. 이 때 웹 사이트 콘텐츠가 지리적으로 사용자와 가까운 CDN 서버에 저장되어 컴퓨터에 훨씬 빨리 도달할 수 있다.
- CDN의 주 목적은 대기 시간을 줄이거나 네트워크 설계로 인해 발생하는 통신 지연을 줄이는 것이다. CDN은 클라이언트와 웹 사이트 서버 간에 중간 서버를 둠으로써 효율성을 높인다.
- CDN 서버는 클라이언트-서버 통신의 일부를 관리한다. 즉, 웹 서버에 대한 웹 트래픽을 줄이고, 대역폭 소비를 줄이며, 애플리케이션의 사용자 환경을 개선한다.
- CDN은 콘텐츠를 저장하고 전송하지만, 웹 애플리케이션의 동적 콘텐츠 처리(예: 로그인 처리, 데이터베이스 접근 등)를 수행하는 웹 서버를 대체할 수는 없다.
  - → 웹 호스트와 동일한 개념이 아니다
- 네트워크 에지에서 콘텐츠를 캐싱하여 웹 사이트 성능을 향상시킨다.

# 주요 용어 및 개념
- 원본 서버
  - 클라이언트가 요청하는 리소스의 원본을 저장하는 서버.
  - 분산 클라우드 프라이빗 링크를 사용하면 프라이빗 링크를 통해 원본 서버를 구성할 수 있다
- 캐싱
  - CDN은 원본 서버보다 최종 사용자에게 물리적으로 더 가까운 프록시 서버에 콘텐츠를 캐싱한다.
  - CDN의 엣지 서버는 사용자의 요청을 수신하여, 캐시된 콘텐츠가 있으면 즉시 제공하고, 없으면 원본 서버로부터 콘텐츠를 가져와 저장한 뒤 제공한다.
  - 서버가 요청하는 사용자와 더 가깝기 때문에 CDN은 콘텐츠를 더 빠르게 전송할 수 있다.
  - 캐싱은 멀리 떨어진 서버들이 쿼리에 응답하여 나중에 사용할 수 있도록 정보를 저장하거나 "캐싱"하는 것을 의미한다.
- 엣지 컴퓨팅
  - 엣지 서버는 사용자와 가까운 네트워크 상 위치에서 컴퓨팅 작업을 수행할 수 있는 장치로, 실시간 처리(예: AI, 게임, 스트리밍 등)에 특화된다. 반면 CDN은 주로 정적 콘텐츠 전송에 초점을 둔다.
  - CDN은 캐시된 데이터 전송에 중점을 두는 반면, 엣지 컴퓨팅은 라이브 스트리밍, 게임, AI 등 다양한 유형의 컴퓨팅도 지원한다.
- PoP(Points of Presence)
  - PoP는 CDN 서비스 제공 지점으로, 전 세계에 분산되어 있으며 각 PoP 간에는 빠른 속도의 전용 네트워크로 연결된다. 이를 통해 사용자에게 빠르고 안정적인 콘텐츠 전달을 보장한다.
  - 이러한 PoP는 여러 Tier1 트랜짓 제공업체와 밀접하게 피어링되어 연결되어 애플리케이션과 소비자에게 안정적인 고품질 인터넷 액세스를 제공한다.

# 이점
1. 웹 사이트 로딩 시간 개선
   - 인근 CDN 서버(기타 최적화 기능 포함)를 사용하여 웹사이트 방문자에게 콘텐츠를 더 가까이 배포함으로써 방문자는 더 빠른 페이지 로딩 시간을 경험할 수 있다.
2. 대역폭 비용 절감
   - CDN은 캐싱 및 기타 최적화를 통해 원본 서버가 제공해야 하는 데이터 양을 줄여 웹사이트 소유자의 호스팅 비용을 절감할 수 있다
3. 콘텐츠 가용성 및 중복성 향상
   - CDN은 분산형 특성 덕분에 여러 원본 서버보다 더 많은 트래픽을 처리하고 하드웨어 장애를 더 잘 견뎌낼 수 있다
4. 웹 사이트 보안 개선
   - CDN은 DDoS 완화, 보안 인증서 개선 및 기타 최적화를 제공하여 보안을 개선할 수 있다.
   - CDN은 또한 웹 애플리케이션 방화벽(WAF)을 제공하여 SQL 인젝션, XSS 공격 등으로부터 보호할 수 있으며, HTTPS 암호화를 통해 안전한 데이터 전송을 지원한다.

# 작동 방식

![](./CDN/image_1.png)

1. 캐싱 (Caching)
   - CDN은 웹사이트의 정적 콘텐츠 (이미지, CSS, JavaScript 등)를 전 세계적으로 분산된 엣지 서버에 복사본 형태로 저장한다.
   - 이렇게 저장된 콘텐츠는 사용자가 웹사이트에 접속할 때, 해당 사용자와 가장 가까운 엣지 서버에서 제공되므로 로딩 시간이 단축된다.
2. 부하 분산 (Load Balancing)
   - CDN은 여러 엣지 서버에 네트워크 트래픽을 분산시켜 서버에 과부하가 걸리는 것을 방지하고, 장애 발생 시에도 서비스 중단 없이 사용자가 웹사이트에 접속할 수 있도록 한다.
3. DNS (Domain Name System) 설정
   - DNS는 사용자가 웹사이트에 접근할 때, CDN의 로직에 따라 요청자의 위치에 가장 가까운 엣지 서버의 IP 주소를 반환한다. 이를 통해 지연 시간을 줄이고 고속 콘텐츠 제공이 가능하다.
